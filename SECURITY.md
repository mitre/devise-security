# Security Policy

## Supported Versions

| Version | Supported          |
|---------|--------------------|
| 0.18.x  | :white_check_mark: |
| < 0.18  | :x:                |

## Reporting a Vulnerability

**Do not open a public issue for security vulnerabilities.**

Instead, please report them via one of:

1. **GitHub Security Advisories** (preferred): [Report a vulnerability](https://github.com/devise-security/devise-security/security/advisories/new)
2. **Email**: Contact the maintainers listed in the [gemspec](devise-security.gemspec)

Please include:

- Description of the vulnerability
- Steps to reproduce
- Affected versions
- Potential impact

## Response

- We will acknowledge receipt within 48 hours
- We will provide an initial assessment within 1 week
- Security fixes are released as patch versions as soon as possible
- We follow [coordinated disclosure](https://en.wikipedia.org/wiki/Coordinated_vulnerability_disclosure) — we ask that you do not publicly disclose until a fix is released

## Scope

This policy covers the `devise-security` gem itself. For vulnerabilities in dependencies (Devise, Rails, etc.), please report to those projects directly.
